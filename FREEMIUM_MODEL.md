# gcc-evo 免增模型设计

> 基本功能免费 | 核心功能收费

---

## 核心原则（一句话）

```
Everyone can use BASIC features for free.
Only customers pay for CORE features.

所有人都能免费用基本功能。
只有付费客户才能用核心功能。
```

---

## 功能分类

### 基本功能 ✅ FREE (所有人)

这些功能让用户体验gcc-evo的价值主张，完全免费：

#### 1. 三层内存（基础版）
```
Sensory Layer (24h)    → 自动记录事件
Short-term (7d)        → 存储决策记录
Long-term (∞)          → 保存验证规则

存储后端：JSON 或 SQLite（本地文件）
成本：零（用户自己的磁盘）
用途：单机使用，完全够用

✅ 任何公司都能免费用
   - 学生、个人开发者
   - 初创公司
   - 大公司（用来测试）
```

#### 2. 混合检索（基础版）
```
三种算法融合：
- 语义相似度 (embedding)
- 时间加权 (recency)
- 关键词匹配 (BM25)

性能：<1秒查询（数千条记录）
成本：零（用户的embedding模型或开源模型）
用途：足以满足基础场景

✅ 任何公司都能免费用
```

#### 3. 经验蒸馏（基础版）
```
通过LLM提炼规则：
- 观察 → 经验卡
- 经验卡 → 技能规则
- 规则 → SkillBank

成本：用户提供自己的API Key（Claude/GPT/Gemini）
用途：基础的知识积累

✅ 任何公司都能免费用
   （只要他们有LLM API）
```

#### 4. Skeptic防幻觉（基础版）
```
置信度门控：
- 高置信(>=0.75) → 自动通过
- 中置信(0.6-0.75) → 请求人工审核
- 低置信(<0.6) → 拒绝

成本：零（纯算法）
用途：防止错误结论写入记忆

✅ 任何公司都能免费用
```

#### 5. 6步Loop闭环（基础版）
```
自动化流程：
1. OBSERVE   - 收集日志
2. AUDIT     - 分析问题
3. EXTRACT   - 提炼规则
4. VERIFY    - Skeptic验证
5. DISTILL   - 写入SkillBank
6. REPORT    - 生成报告

成本：零（编排逻辑）
周期：自定义（5分钟、1小时、1天）

✅ 任何公司都能免费用
```

**总结基本功能**：
- 用户可以构建**完整的、可工作的系统**
- 可以体验gcc-evo的全部价值主张
- 足以满足**大多数中小企业的需求**
- **对大公司也足以测试和评估**

### 核心功能 💰 PAID (仅付费客户)

这些功能是生产级别的，只有付费客户才能获得：

#### 1. 分布式存储

**基础版缺陷**：
```
JSON/SQLite 都是本地文件
├─ 无法跨服务器同步
├─ 无法实时复制
├─ 无法高可用部署
├─ 单点故障风险
└─ 扩展到数百万记录时缓慢
```

**企业版解决**：
```
分布式存储：
├─ Redis  - 实时数据
├─ PostgreSQL - 持久化
├─ Kafka - 消息队列
└─ S3 - 冷数据存储

特性：
✓ 主从复制（高可用）
✓ 自动分片（横向扩展）
✓ 跨地域同步（容灾）
✓ 实时一致性（强保证）

成本：需要基础设施 + 维护
价格：$1,000+/月
```

#### 2. 多用户协作

**基础版缺陷**：
```
单用户系统
├─ 无法共享状态
├─ 无法协作编辑
├─ 无法权限隔离
└─ 无法审计日志
```

**企业版解决**：
```
多用户协作：
├─ 用户管理 (RBAC)
├─ 权限控制 (read/write/admin)
├─ 实时同步 (WebSocket)
├─ 冲突解决 (OT/CRDT)
├─ 审计日志 (完整记录)
└─ 团队工作区

成本：复杂的并发处理
价格：$1,000+/月
```

#### 3. 企业集成

**基础版缺陷**：
```
独立系统
├─ 无法连接Kafka
├─ 无法暴露gRPC/REST API
├─ 无法与企业系统通信
└─ 无法订阅外部事件
```

**企业版解决**：
```
企业级集成：
├─ Kafka connector (消息流)
├─ gRPC API (高性能RPC)
├─ REST API (标准HTTP)
├─ Webhook支持 (事件驱动)
├─ OAuth/SAML (企业认证)
└─ VPC部署 (网络隔离)

成本：复杂的集成开发
价格：$1,000+/月
```

#### 4. 高级检索优化

**基础版缺陷**：
```
通用实现
├─ 无法针对大规模优化
├─ 无法自定义embedding模型
├─ 无法智能缓存
└─ 数百万记录时变慢
```

**企业版解决**：
```
优化版本：
├─ 自训练embedding模型 (针对行业数据)
├─ 分层索引 (快速筛选)
├─ 智能缓存 (热数据预热)
├─ 向量量化 (加速匹配)
└─ 异步处理 (吞吐量优化)

成本：专有算法 + 优化工程
价格：$1,000+/月
```

#### 5. SLA保证 + 专业支持

**基础版缺陷**：
```
开源项目，无保证
├─ 无法保证可用性
├─ 无法保证响应时间
├─ 无法提供电话支持
└─ 出问题自己调试
```

**企业版解决**：
```
企业级支持：
├─ 99.9% uptime SLA
├─ 4小时响应时间
├─ 24/7电话支持
├─ 专属技术账户经理
├─ 优先级修复
└─ 定期审查会议

成本：运维 + 支持人员
价格：$1,000+/月
```

## 实现方式

### 代码结构

```
gcc_evolution/
├─ core/
│  └─ pipeline.py          # 框架，MIT开源，所有人用
│
├─ memory/
│  ├─ basic_tier.py        # ✅ 基础版 (BUSL)
│  │  └─ FileStorage (JSON/SQLite)
│  │
│  └─ enterprise_tier.py    # 💰 企业版 (私有)
│     └─ DistributedStorage (Redis/PG)
│
├─ retrieval/
│  ├─ basic_retriever.py    # ✅ 基础版 (BUSL)
│  │  └─ BM25 + embedding + temporal
│  │
│  └─ enterprise_retriever.py # 💰 企业版 (私有)
│     └─ Advanced: custom models + caching
│
├─ distillation/
│  └─ distiller.py          # ✅ 基础版 (BUSL)
│     └─ LLM-based extraction
│
├─ decision/
│  └─ skeptic.py            # ✅ 基础版 (BUSL)
│     └─ Confidence gate
│
├─ orchestration/
│  └─ loop_engine.py        # ✅ 基础版 (BUSL)
│     └─ 6-step closure
│
└─ enterprise/              # 💰 企业版 (私有代码)
   ├─ collaboration/
   ├─ integration/
   ├─ observability/
   └─ scaling/
```

### 初始化时的选择

```python
# 用户初始化时选择
loop = GCCEvo.init(
    storage_type='file',      # ✅ 免费版
    # OR
    storage_type='redis',     # 💰 需要授权
)

# 基础功能
loop.run_with_basic_features()  # ✅ 始终可用

# 企业功能
loop.run_with_enterprise_features()
# ↑ 需要企业授权
# ↑ 如果无授权，会提示购买或升级
```

## 与许可证的关系

```
Free版本 (基本功能)
  ├─ Tier 1 (MIT)        → 框架基础，所有人自由用
  ├─ Tier 2 (BUSL)       → 核心实现，开源可用
  └─ License: BUSL 1.1   → 商用可用（基础版范围内）

Enterprise版本 (核心功能)
  ├─ Tier 3 (Proprietary) → 企业版，代码不开源
  └─ License: Custom      → 企业许可证
```

## 为什么这个模型好

### 对用户的好处
✅ 清晰的分界 - 明确知道哪些免费哪些收费
✅ 低风险尝试 - 免费版足以评估价值
✅ 可逐步升级 - 从免费版升到企业版
✅ 无强制 - 基本版完全可用，不受限

### 对你的好处
✅ 清晰的收费点 - 只有核心功能收费
✅ 大用户基数 - 所有人都能试用基本版
✅ 自然转化 - 需要扩展时自动升级
✅ 可持续收入 - 企业版稳定付费

### 对社区的好处
✅ 开源友好 - 基础功能开源可用
✅ 生态鼓励 - 任何人可在基础上构建
✅ 长期信任 - 透明的收费模型
